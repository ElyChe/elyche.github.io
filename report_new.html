  ⚠️ Attainable Up < Sync Up — данные несогласованны, но в любом случае линия критически деградировала.
Рекомендации 1. Домашняя часть: сплиттер обязателен, убрать все параллельные телефоны/удлинители/скрутки, проверить ввод и контакты.
2. Провайдеру: измерения пары тестером (TDR/SELT) для локализации повреждения — вероятен обрыв/замыкание на высоких частотах Up; смена пары/кросса, проверка порта.
3. При невозможности починки: миграция на VDSL/FTTB, если доступно.
Что ещё запросить Обязательно: CRC/FEC, ES/SES, LOS/LOF, retrain count, TDR/SELT-тест пары (изоляция, сопротивление, ёмкость, локализация повреждения), сведения о профиле/DLM.
Что может бытьОбрыв/замыкание пары на участке абонент–кросс: Up практически отсутствует, высокое затухание 37.6 dB. Типичные причины — окисление скруток, влага в муфте, повреждение кабеля.
Минимальный набор для точной диагностики
  • Счётчики ошибок и событий: CRC/FEC, ES/SES, LOS/LOF, retrain count, uptime (желательно за 24–72 часа).
  • Параметры защиты/профиля: G.INP/INP, interleave depth/delay, SRA/bit-swap, vectoring (если доступно).
  • Домашняя часть: один прямой ввод до модема, без ответвлений; корректный сплиттер/фильтры (особенно для ADSL/телефонии).
Справка: пороги SNR Margin
  • ≥15 dB — отличный запас, линия стабильна.
  • 10–15 dB — нормальный запас, при низких ошибках линия работоспособна.
  • 6–10 dB — низкий запас, риск ошибок при помехах.
  • <6 dB — критический, высока вероятность ретрейнов и ошибок.
⚠️ Валидация данных
  • Если Sync > Attainable — данные несогласованны (сняты в разное время или ошибка источника). Требуется повторный замер.
  • Если Attn = 0 или пустое — устройство не отдало данные; нельзя оценить длину линии.
  • Нестандартные режимы («etsi», «ADSL2+ (17A)») — вероятна ошибка парсинга или источника.
