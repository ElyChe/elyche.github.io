xDSL — краткий отчёт по линиям

В отчёте нет счётчиков ошибок (CRC/FEC/ES/SES/LOS), поэтому ниже — оценка текущей «физики» (SNR/затухание/скорости) и что запросить для точной диагностики.

Отлично
VDSL2 (Annex B)

Attainable (Down/Up)92.9 / 47.5 Mbps
SNR (Down/Up)20.0 / 28.6 dB
Attn (Down/Up)9.3 / 16.9 dB
НаблюдениеUp заметно ниже attainable → вероятно ограничение профилем/тарифом.
РекомендацииПри запросе «больше Up» — сверить тариф/профиль, наличие vectoring/G.INP.
Что ещё запроситьОпционально: CRC/FEC, ES/SES, LOS/LOF, retrain count, uptime за 24–72ч.
Что может бытьПрофиль/тариф ограничивает Up: Sync 10.24 при attainable 47.5 и высоком SNR; при низких CRC/ES линия физически отличная.

Наблюдение ⚠️ Высокое затухание + низкие скорости → длинный участок/деградация меди/плохая домашняя разводка.

РекомендацииУпростить домашнюю линию до модема (одна пара без ответвлений), проверить розетку/контакты/кабель. Провайдеру — проверка пары/кросса/порта, возможная смена пары.
Что ещё запросить Нужно: CRC/FEC, ES/SES, retrain count, LOS/LOF, график SNR по времени, измерения пары (изоляция/сопротивление/ёмкость).
Что может бытьДлинный/слабый участок или плохая медь: высокое затухание 31/37 dB и низкий sync; улучшение будет при исправлении пары/домовой разводки.

  ⚠️ Attainable Up < Sync Up (несостыковка источника).
Рекомендации Срочно исключить домашние причины: сплиттер обязателен, убрать параллельные телефоны/удлинители/скрутки, проверить ввод/контакты. Затем провайдеру: поиск плохого контакта/окисления/влаги, смена пары/кросса, проверка порта; при возможности — миграция на VDSL/FTTx.
Что ещё запросить Нужно: CRC/FEC, ES/SES, LOS/LOF, retrain count, измерения пары (изоляция/сопротивление/ёмкость), сведения о профиле/DLM.
Что может бытьДеградация пары/контактов: Up 0.093 и высокий Attn 37.6; при росте CRC/ES и ретрейнов нужна проверка линии и смена пары.
Минимальный набор для точной диагностики
  • Счётчики ошибок и событий: CRC/FEC, ES/SES, LOS/LOF, retrain count, uptime (желательно за 24–72 часа).
  • Параметры защиты/профиля: G.INP/INP, interleave depth/delay, SRA/bit-swap, vectoring (если доступно).
  • Домашняя часть: один прямой ввод до модема, без ответвлений; корректный сплиттер/фильтры (особенно для ADSL/телефонии).
